# ADR-0001：企业框架优先（不从零造轮子）

- 状态：已采纳
- 日期：2026-06-30

## 背景
小念要做到企业级、可被企业直接应用，同时规避"自研不靠谱""抄袭风险"两个问题。

## 决策
底层全部站在大厂在生产验证过的成熟开源框架上（以 pip 依赖方式引入），
我们只写产品逻辑（人格、关心策略、记忆结构、技能、编排粘合）。用依赖 ≠ 抄源码。

选型：
- 编排：LangGraph（Klarna / Uber / 摩根大通在生产使用；持久执行 + 人在环 + 状态持久化）
- 记忆：Mem0（业界领先的 Agent 记忆层，亚秒级，可本地）
- 技能：Anthropic Agent Skills 开放标准（SKILL.md + 三级渐进式披露）
- 电脑操控：内置受控 harness（默认），Open Interpreter 作为可选进阶后端
- UI：本地 Web（assistant-ui 思路；本机零 Node 构建版本以 FastAPI 托管静态前端）
- 模型：多 provider，全 OpenAI 兼容（MiniMax / DeepSeek / 小米 MiMo）

## 结果
- 正：起点高、可维护、可被企业复用；每个技术点都有大厂背书。
- 负：引入外部依赖的版本/兼容性约束（见 ADR-0003 关于 Python 版本）。
